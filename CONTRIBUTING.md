## Getting Help

Before asking a question:  

1. Search the [issue tracker](https://github.com/wdkqqq/rift/issues) to see if it has already been answered.  
2. Check online resources (Stack Overflow, forums, blogs).  

If your question still needs clarification:  

- Open a new [Issue](https://github.com/wdkqqq/rift/issues/new).  
- Provide context: code, environment, versions, and steps to reproduce.  
- Describe what you expected vs. what actually happened.  

---

#### Preparing Your Report

A well-documented bug report helps us fix issues faster:  

- Make sure you are using the latest version.  
- Confirm the issue is not caused by your environment.  
- Search for existing issues first.  
- Collect all relevant details:  
  - Stack trace or error logs  
  - Operating system, platform, architecture  
  - Versions of relevant tools (Node.js, npm, etc.)  
  - Steps to reproduce the problem  
  - Input and output examples  

#### Submitting Your Report

- Open a new [Issue](https://github.com/wdkqqq/rift/issues/new).  
- Include:  
  - Expected behavior  
  - Actual behavior  
  - Reproduction steps  
  - Screenshots or GIFs if applicable  

> Security issues should **never** be posted publicly. Email them to wdkq@bladerunn.in.  

---

#### Before Submitting

- Check that the feature doesn’t already exist.  
- Make sure it benefits most users, not just a niche case.  
- Search the issue tracker for similar suggestions.  

#### Submitting Your Suggestion

- Open a clear, descriptive [Issue](https://github.com/wdkqqq/rift/issues/new).  
- Describe:  
  - Current behavior  
  - Proposed change  
  - Step-by-step instructions or examples  
  - Why it’s useful  
- Screenshots, GIFs, or references are encouraged.  

---

### Code Contributions

- Set up your development environment according to [project instructions](https://github.com/wdkqqq/rift/blob/main/README.md#build-from-source)  
- Fork the repository and create a new branch for your changes  
- Make small, focused commits with descriptive messages  
- Include tests where appropriate  
- Submit a pull request (PR) and link related issues  

---

## Coding & Style Guidelines

- Follow existing code style and formatting  
- Write clear, concise, and maintainable code  
- Comment where necessary but avoid obvious comments  
- Keep commits focused and descriptive  

### Commit Messages

- Use present context: `feat:`, `fix:`, `chore:`  
- Include a short summary and optional detailed description  
- Reference related issues when relevant: `Fixes #123`
